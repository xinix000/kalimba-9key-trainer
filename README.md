# Kalimba 9 คีย์ — เว็บฝึกเล่น

เว็บสอนเล่นคาลิมบาแบบโน้ตร่วง (falling notes) ทำมาให้ตรงกับเครื่อง **NuNu mini 9 คีย์**
โดยเฉพาะ เพราะแอปในสโตร์ส่วนใหญ่รองรับแค่ 17 กับ 21 คีย์

เปิดจากคอม แล้วให้มือถือในวง Wi-Fi เดียวกันเข้ามาดูได้ ถือคาลิมบาจริงเล่นตามได้เลย

## เริ่มใช้

```
python server.py
```

โปรแกรมจะพิมพ์ URL ออกมา 2 อัน

```
บนคอมเครื่องนี้ :  http://localhost:8000
บนมือถือ        :  http://192.168.x.x:8000
```

พิมพ์อันล่างลงมือถือ (ต้องต่อ Wi-Fi วงเดียวกับคอม) ถ้าเข้าไม่ได้ให้เปิดพอร์ตใน
Windows Firewall ตามคำสั่งที่โปรแกรมพิมพ์ไว้ ใช้แค่ Python มาตรฐาน ไม่ต้อง `pip install`

เปลี่ยนพอร์ตได้ด้วย `python server.py --port 8080`

## ผังลิ้น

อ่านจากเลขที่สลักบนลิ้นจริง เรียงซ้ายไปขวา

| ตำแหน่ง | 1 | 2 | 3 | 4 | 5 (กลาง) | 6 | 7 | 8 | 9 |
|---|---|---|---|---|---|---|---|---|---|
| เลขบนลิ้น | 5 | 3 | 1 | 6· | **5·** | 7· | 2 | 4 | 6 |
| โน้ต | G4 | E4 | C4 | A3 | **G3** | B3 | D4 | F4 | A4 |

ลิ้นกลางยาวที่สุดคือเสียงต่ำสุด แล้วไล่สลับซ้าย-ขวาออกไป
เรียงตามระดับเสียงได้เป็น `5· 6· 7· 1 2 3 4 5 6` (C major ตั้งแต่ G ต่ำถึง A สูง)

คอลัมน์โน้ตบนจอตรงกับตำแหน่งลิ้นจริง ดูว่าโน้ตร่วงลงช่องไหนก็ดีดลิ้นนั้น

## โหมด

| โหมด | ทำอะไร |
|---|---|
| ฟังก่อน | เว็บเล่นให้ฟัง ลิ้นบนจอสว่างตามจังหวะ ใช้ดูให้จำก่อน |
| เล่นตาม | เว็บเงียบ (เหลือแต่เมโทรนอม) โน้ตร่วงลงมาให้ดีดคาลิมบาจริงตาม |
| รอฉัน | หยุดรอทุกตัวจนกว่าจะแตะลิ้นให้ถูกบนจอ ไม่มีจำกัดเวลา |

ปรับความเร็วได้ 0.5× ถึง 1.5× (ปุ่มกลางกดกลับ 1.0×) เปิด/ปิดเมโทรนอมและเล่นวนซ้ำได้

ปุ่ม **อ็อกเทฟ** ใช้ปรับเสียงในเว็บให้ใกล้เคียงเครื่องจริง ตัว mini เสียงสูงกว่าคาลิมบา
ตัวเต็มไซซ์ประมาณหนึ่งช่วงคู่แปด ค่าเริ่มต้นตั้งไว้ที่ "มินิ" แล้ว ถ้าฟังแล้วไม่ตรงลองสลับดู

เปิดบนคอมกดคีย์ `1`–`9` แทนลิ้นซ้ายไปขวาได้ และ `space` = เล่น/หยุด

## แนวตั้ง / แนวนอน

หมุนจอได้ทั้งสองแบบ เลย์เอาต์สลับให้เอง ไม่ต้องรีเฟรช

- **แนวตั้ง** โน้ตร่วงเต็มความกว้าง ปุ่มควบคุมเรียงอยู่ด้านล่าง
- **แนวนอน** ปุ่มย้ายไปแถบข้างขวาทั้งหมด โน้ตร่วงจึงได้ความสูงเต็มจอ
  แถบโน้ตตัวเลขกลายเป็นตารางหลายบรรทัด มองเห็นเพลงทั้งเพลงพร้อมกัน

แนวนอนเหมาะกับตอนวางมือถือพิงอะไรไว้แล้วถือคาลิมบาสองมือ ส่วนแนวตั้งเหมาะกับตอนถือมือถือ

## เพิ่มเพลงเอง

### วิธีที่ 1: พิมพ์ในหน้าเว็บ (ง่ายสุด ทำบนมือถือได้)

กดปุ่ม ✎ บนหัวเว็บ ใส่ชื่อเพลง BPM จังหวะต่อห้อง แล้วพิมพ์โน้ตจากหนังสือลงไป
ระบบจะบอกทันทีว่าอ่านได้กี่ตัว กี่ห้อง และมีโน้ตไหนที่เครื่อง 9 คีย์เล่นไม่ได้บ้าง
กด "ลองฟัง" ฟังก่อนบันทึกได้ เพลงที่บันทึกเก็บไว้ในเบราว์เซอร์ตัวนั้น
(เปิดจากคอมกับจากมือถือจะเป็นคนละชุด)

เลือกเพลงที่พิมพ์เองไว้แล้วกด ✎ อีกครั้ง = แก้หรือลบเพลงนั้น

### วิธีที่ 2: แก้ไฟล์

แก้ `web/songs.js` แล้วรีเฟรชหน้าเว็บ (เซิร์ฟเวอร์ปิดแคชไว้ให้แล้ว ไม่ต้องรีสตาร์ท)

```js
{
  id: 'mysong',
  title: 'ชื่อเพลง',
  subtitle: 'คำอธิบายสั้นๆ',
  bpm: 100,
  beatsPerBar: 4,
  startBeat: 0,        // ใส่ค่าลบถ้ามีห้องยก เช่น -1
  seq: `
    1 1 5 5 | 6 6 5:2
  `,
}
```

กติกาการเขียน `seq`

| เขียนแบบนี้ | ในหนังสือคือ | หมายความว่า |
|---|---|---|
| `1` | `1` | โน้ต 1 ยาว 1 จังหวะ |
| `5.` | `5` มีจุดข้างใต้ | เสียงต่ำ พิมพ์จุดไว้ข้างหลังแทน |
| `3:0.5` | `3` มีขีดใต้ 1 เส้น | ครึ่งจังหวะ (ขีดสองเส้น = `:0.25`) |
| `1:1.5` | `1·` | โน้ตประจุด |
| `-` | `-` | ลากเสียงตัวก่อนหน้าต่ออีก 1 จังหวะ |
| `0` | `0` | หยุด (`0:2` = หยุด 2 จังหวะ) |
| `\|` | `\|` | เส้นกั้นห้อง ใส่ไว้อ่านง่ายเฉยๆ ระบบไม่สนใจ |

เขียนตามหนังสือได้เกือบตรงตัว ต่างกันแค่จุดอ็อกเทฟย้ายมาไว้ข้างหลัง
กับความยาวโน้ตที่พิมพ์เป็นตัวเลขแทนขีดใต้

โน้ตที่ใช้ได้มีแค่ `5. 6. 7. 1 2 3 4 5 6` ถ้าใส่โน้ตที่เครื่องไม่มี ตัวแก้ไขในหน้าเว็บ
จะบอกทันทีว่าตัวไหน (ถ้าแก้ในไฟล์จะเตือนใน console ของเบราว์เซอร์แทน)

เพลงไหนเสียงเกินช่วงนี้ต้องย้ายคีย์ (transpose) ก่อน หรือยกโน้ตที่เกินขึ้น/ลงหนึ่งอ็อกเทฟ

```

web/index.html     โครงหน้า
web/styles.css     ธีม
web/songs.js       ผังลิ้น + คลังเพลง  <- แก้ตรงนี้เวลาเพิ่มเพลง
web/audio.js       สังเคราะห์เสียงคาลิมบาด้วย Web Audio (ไม่ต้องมีไฟล์เสียง)
web/app.js         โน้ตร่วง จังหวะ และการควบคุมทั้งหมด
```
